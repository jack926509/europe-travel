# 🇪🇸🇵🇹🇫🇷 西班牙・葡萄牙・南法 自由行全攻略 2025-2026

這是一份專為手機端優化的歐洲自由行網頁應用程式（Web App），目標是作為旅行當下的隨身資料庫，包含行前準備、交通票券、景點美食、購物退稅等完整資訊。

---

## 👨‍💻 三角色優化檢視與建議

### 1️⃣ 自由行旅遊西歐達人視角
**檢視結果：** 
旅遊資料非常詳實，涵蓋了背包客與自由行旅客最需要的痛點（如治安防竊、罷工應對、跨國交通限制、以及退稅門檻等）。特別是 ETIAS 和 EES 的提醒非常及時！將「貼士」改為「叮嚀」更符合台灣人的語境，讓人感覺更親切與關心。將警察與急救電話放在最後一區是極佳的安排。
**優化與建議：**
- **已優化**：字眼調整為「叮囑/叮嚀」，不僅統一且語調溫和；縮短了導航標籤（如「景點行程」縮為「景點」），在趕路查資料時能一眼掃過不吃力。
- **未來建議**：可以考慮加上「日出/日落時間」的 API，歐洲夏冬日照時間差異極大，對排行程影響很深。另外可考慮補充「公廁分佈或尋找技巧」，這是歐洲旅遊的一大痛點。

### 2️⃣ 前端工程師視角
**檢視結果：** 
作為一個「給自己在手機上作為旅遊資料庫」的專案，原本單頁式（Single Page）超過 4000 行的設計在讀取跟視覺尋找上負擔太大，很容易滑到迷失方向。
**優化與建議：**
- **已優化（本次核心亮點）**：
  1. **分頁系統架構 (Pagination)**：實作了 5 個核心分頁（行前、交通、攻略、購物、叮囑），透過隱藏/顯示區塊達到毫秒級的切換速度。
  2. **底部導航列 (Bottom Navigation)**：採用 iOS 質感的毛玻璃效果設計（`backdrop-filter: blur`），加入圖標，讓在大拇指可及範圍內單手操作。
  3. **標籤自適應縮小 (Compact Tabs)**：將各子分頁按鈕（如交通和迪士尼的選項）縮減 padding 與字數（如 `跨國移動` -> `跨國`），並加上 `overflow-x: auto`，順暢滑動不吃畫面寬度。
  4. **色彩與排版升級**：調整主色調（Terra & Saffron），讓視覺更沈穩高雅（深琥珀/陶土紅），字體粗細與行距 (`line-height: 1.8`) 調大，強光下看手機也不吃力。
- **未來建議**：若資料量持續擴增，建議拆分成多個 JSON 檔或導入本地 IndexedDB，搭配簡單的 Vue 或是 React 來管理元件，可大幅提高後續擴充的維護性。可以加上 PWA (Progressive Web App) manifest，讓網頁能真正「安裝」到手機桌面上，並且支援離線快取瀏覽。

### 3️⃣ 後端工程師視角
**檢視結果：** 
目前為純靜態 HTML/CSS/JS 檔案，好處是不用伺服器即可瀏覽（非常適合出國沒網路時看下載好的頁面），但所有內容寫死在 HTML 內，維護成本較高。
**優化與建議：**
- **已優化**：重構了 JavaScript 部分的函數 (`switchPage`, `showTabPanel`)，統一將 DOM 操作集中管理。增加了動態監測 Resize 的函式，確保螢幕寬度改變時（如轉到桌面版）不會發生版面消失的 Bug。
- **未來建議**：建議將龐大的景點與美食數據結構化（轉存為 JSON），未來若要架構後台（例如使用 Python FastAPI 或 Node.js）來動態新增餐廳或匯率，可以直接串接 API。

## 📝 修改歷程 (Changelog)

### v2.1 ｜ 2026-03-11 (Mobile UX/UI 全面進化)
*   **介面架構重構**：將一頁式超長版面轉換為「5大分頁系統」，支援無縫切換，大幅減少滾動疲勞。
*   **新增底部導航列 (Tab Bar)**：新增如原生 App 般的底部導航列，具備毛玻璃與高亮選取效果，支援大拇指快速切換。
*   **手勢操作支援 (Swipe)**：增加左右滑動（Swipe）手勢以切換主分頁，提升手機使用者體驗。
*   **版面微調與壓縮**：
    *   將過長的子導航按鈕精簡化（例如：「跨國移動」縮短為「跨國」、「景點行程」縮為「景點」）。
    *   縮小按鈕 Padding 與間距，防止在小螢幕手機上破版。
*   **文字語氣優化**：將整體的「貼士」全數調整為「叮嚀/叮囑」，更符合台灣使用者習慣。
*   **色彩與字體加強**：更換整體色系為更具質感的暖色系，增大手機版預設行距。
*   **頁尾 (Footer) 優化**：新增更新版本號、免責聲明、以及重要的 ETIAS/EES 首發預警資訊。
